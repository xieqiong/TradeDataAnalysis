题目A、大文件处理
题目描述
请编写一个程序，不停地从某个文本文件(1000万+成交记录，字段包括：交易所代码/交易编码/合约代码/买卖方向/成交价格/成交数量/成交编号等)读取消息然后按要求处理，为了保证程序意外中止后仍能继续执行，消息必须考虑先进行缓存，然后再处理，处理结果写到本地另一个文本文件。
处理要求： 
1）统计每个交易编码在每个合约上的持仓量。
2）从1000万条成交数据中，找出自成交的所有成交组合（成交编号及交易编码相同的成交组）
3）使用附件中的数据进行测试（该数据为交易系统的IncTrade表里的数据）
4）数据处理的过程中，手动终止该程序，再重新启动程序后可以利用缓存接着处理
5）结果提交时，除了代码和执行程序之外，需有文档简要说明处理思路
